# 2026-06-12 — Initial build

First full build session. Went from a freshly flashed SD card to a working offline survival server with knowledge base, document library, maps, and a self-hosted hotspot.

## OS + access

- Flashed Raspberry Pi OS (Desktop) to a 64GB microSD via Raspberry Pi Imager, with SSH and WiFi configured in the **Services** / **General** tabs.
- First boot done on a monitor to enable SSH (`sudo systemctl enable --now ssh`).
- Connected over SSH from the main workstation.
- Set a static IP in `/etc/dhcpcd.conf` so the address stays put.

## Base packages

```bash
sudo apt update && sudo apt install -y kiwix-tools nginx python3 python3-pip git curl wget
```

## Knowledge base (Kiwix)

- Confirmed `kiwix-serve` installed via apt (`/usr/bin/kiwix-serve`).
- Downloaded ZIM files into `~/kiwix/zims/`:
  - `wikipedia_en_simple_all_maxi_2026-05.zim` — Simple English Wikipedia
  - `wikipedia_en_medicine_maxi_2026-04.zim` — WikiMed medical encyclopedia
  - `zimgit-post-disaster_en_2024-05.zim` — post-disaster survival wiki
- Created a systemd service `kiwix.service` to auto-start the server.
  - **Gotcha:** systemd doesn't expand shell wildcards in `ExecStart`. Wrapped the command in `bash -c '...'` so the `*.zim` glob works.
  - **Gotcha:** when proxied behind Nginx at `/kiwix/`, the library page loaded blank because Kiwix served assets from root paths. Fixed with `--urlRootLocation /kiwix` and pointing the Nginx `proxy_pass` at `http://localhost:8888/kiwix/`.

## Web homepage (Nginx)

- Single landing page at `/var/www/html/index.html` linking to Knowledge Base, Maps, and Documents.
  - **Gotcha:** emojis rendered as mojibake until `<meta charset="UTF-8">` was added.
- Nginx config serves the homepage and reverse-proxies `/kiwix/`, plus `autoindex on` for `/docs/`.

## Document library

- Created `/var/www/html/docs/` with directory listing enabled.
- Pushed 70+ survival/military/medical/emergency PDFs from the workstation via `scp`.
  - Includes FM 21-76, Where There Is No Doctor/Dentist, FEMA & DoD prep guides, land-nav manuals, cold-weather handbooks, bug-out checklists, etc.
  - **Note:** browser ad-blocker (uBlock / Brave Shields) threw `ERR_BLOCKED_BY_CLIENT` opening some files — client-side only, server was fine. Disabled shields for the local IP.

## Offline maps

- Approach: **PMTiles + MapLibre GL JS**, served as static files — no tile-server daemon (keeps RAM free on the 1GB Pi).
- Extracted the Eastern US from the Protomaps daily build using `go-pmtiles`:
  ```bash
  ./pmtiles extract https://build.protomaps.com/<date>.pmtiles eastern-us.pmtiles --bbox=-90.0,24.5,-66.9,47.5
  ```
- `scp`'d `eastern-us.pmtiles` to `/var/www/html/maps/`.
- Hosted MapLibre + PMTiles libs locally so the page works fully offline.
- Upgraded from a hand-rolled basic style to the official **Protomaps basemap** (`@protomaps/basemaps` + `basemaps-assets` for fonts/sprites) for proper styled roads, water, labels, and a scale bar.

## GPS (staged, no hardware yet)

- Installed `gpsd` + `gpsd-clients`, set `USBAUTO="true"` so a dongle auto-detects.
- Wrote `gps-bridge.py` — reads gpsd over its socket and exposes the latest fix as JSON at `127.0.0.1:8889`, proxied by Nginx at `/gps`.
- Added a **Locate Me** button + live position marker to the map page.
- Verified `curl http://192.168.1.236/gps` returns `{"fix": false}` — full chain staged, plug-and-play once a VK-172 / BU-353 arrives.

## Self-hosted hotspot (dual-interface)

- Installed `hostapd` + `dnsmasq`.
- Plugged in a TP-Link 802.11ac USB dongle — confirmed **AP mode** support via `iw list` and that it came up as `wlan1`.
- Configured the hotspot on **wlan1** so the built-in **wlan0** stays on the home network (SSH never drops). No more mode-switching scripts.
  - `hostapd.conf` → `interface=wlan1`, SSID `SurvivalKit`, WPA2.
  - `dnsmasq.conf` → `interface=wlan1`, DHCP range `192.168.x.x–20`, gateway `192.168.x.x`.
  - `wlan1-setup.service` → assigns static `192.168.x.x` to wlan1 on boot before hostapd.
  - `denyinterfaces wlan1` in `dhcpcd.conf` so dhcpcd leaves it alone.
- Enabled all services to start on boot.
  - **Gotcha:** first phone test hung because dnsmasq was still running its pre-edit config — a restart fixed it. (And one false alarm where I simply hadn't joined the SurvivalKit network yet.)
- Confirmed: phone connects to `SurvivalKit`, browses to `http://192.168.x.x`, and homepage / Kiwix / docs / maps all load — all while SSH stays alive on wlan0.

## State at end of session

Working: knowledge base, document library, maps, hotspot. GPS staged. Build is functionally complete for offline use.
